# Muscle Beat 2 - MVP仕様書

## 概要

**Muscle Beat 2** は、EDM音楽に合わせてリズムゲーム形式で筋トレを行うiOSネイティブアプリ。
トレーニング成果に応じてキャラクターが育成される要素を持つ。

| 項目 | 内容 |
|------|------|
| プラットフォーム | iOS（初期リリース） |
| 技術スタック | React Native |
| 通信方式 | オフラインファースト（サーバー不要） |
| データ保存 | ローカルDB（SQLite / AsyncStorage） |
| 認証 | なし（ログイン不要） |
| 音楽 | プリセット曲（アプリにバンドル） |

## 画面構成

### 1. ホーム画面

- アプリ起動時に最初に表示される画面
- 育成キャラクターがアニメーション付き（待機モーション）で表示される
- キャラクターの現在の進化段階・経験値が確認できる
- 主要ナビゲーション:
  - トレーニング開始（曲選択画面へ）
  - トレーニング記録（カレンダー画面へ）

### 2. 曲選択画面

- プリセット曲一覧を表示（MVP時点で3曲）
- 各曲カードに表示する情報:
  - 曲名
  - サムネイル
  - 対象トレーニング種目（スクワット / 腹筋 / バーピー）
  - 対象部位（脚 / 腹 / 全身）
- 難易度切替: Normal / Hard
- 曲を選択するとゲーム準備画面へ遷移

### 3. ゲーム準備画面

- 選択した曲名・種目名を表示
- スマホの推奨持ち方をイラスト付きでガイド表示
  - 全種目共通: スマホは目の前で手に持つ
- 「START」ボタンでゲームプレイ画面へ遷移

### 4. ゲームプレイ画面

- **レイアウト構成:**
  - 上部: 降ってくるノーツ（音符マーカー）
  - 下部: ターゲットライン（判定ゾーン）
  - 背景: 運動フォームのアニメーション表示
  - スコア表示（リアルタイム）
  - コンボ表示

- **操作方法:**
  - スマホを手に持ち、運動の動きによる加速度センサー（DeviceMotion）で判定
  - 動きのしきい値を超えた時点で判定処理が発火

- **判定システム:**

  | 判定 | スコア | 許容タイミング |
  |------|--------|--------------|
  | Perfect | 1000 | ±50ms |
  | Great | 500 | ±120ms |
  | Good | 100 | ±265ms |
  | Miss | 0 | ±310ms超 |

- **ゲーム仕様:**
  - 曲の長さ: 約1分
  - ノーツデータ: JSON形式のビートマップ（targetTime, track情報）
  - 音楽再生と同期してノーツが降下
  - ノーツ到達タイミングと運動動作のタイミングで判定

### 5. リザルト画面

- 表示内容:
  - トータルスコア
  - 判定内訳（Perfect / Great / Good / Miss 各回数）
  - 最大コンボ数
  - 獲得経験値
  - キャラクターの経験値バー（進化までの進捗）
- アクション:
  - ホームへ戻る
  - リトライ（同じ曲を再プレイ）

### 6. トレーニング記録画面（カレンダー）

- カレンダー形式でトレーニング履歴を表示
- トレーニングした日にマーカー表示
- 日付タップで詳細表示:
  - プレイした曲名
  - 種目
  - スコア
  - 獲得経験値

## コア機能

### A. リズム筋トレゲーム

#### トレーニング種目（MVP: 3種目）

| 種目 | 対象部位 | 動きの特徴 | センサー検知 |
|------|---------|-----------|------------|
| スクワット | 脚 | 規則的な上下動 | Y軸加速度の周期的変化 |
| 腹筋（クランチ） | 腹 | 上体の起き上がり | Z軸の傾き変化 |
| バーピー | 全身 | 立つ→しゃがむ→ジャンプ | 大きなY軸加速度変化 |

#### ビートマップ仕様

```json
{
  "songId": "song_001",
  "bpm": 128,
  "offset": 0,
  "difficulty": "normal",
  "notes": [
    {
      "targetTime": 1500,
      "track": 1,
      "stroke": "#FF00FF"
    }
  ]
}
```

- `targetTime`: 曲開始からの経過時間（ミリ秒）
- `track`: ノーツのレーン（視覚的な表示位置）
- `stroke`: ノーツの色
- `offset`: 音楽再生とノーツのタイミング補正値

#### 曲構成（MVP: 3曲）

| # | 曲 | 種目 | 難易度 |
|---|-----|------|--------|
| 1 | 曲A | スクワット | Normal / Hard |
| 2 | 曲B | 腹筋 | Normal / Hard |
| 3 | 曲C | バーピー | Normal / Hard |

※ 各曲約1分。曲データ（MP3）とビートマップ（JSON）はアプリにバンドル。

#### モーションセンサー仕様

- `expo-sensors` の `Accelerometer` を使用
- サンプリングレート: 60Hz
- 判定しきい値: 加速度の絶対値が一定値を超えた場合にトリガー
  - 種目ごとにしきい値と検知軸を調整
- クールダウン: 連続誤検知防止のため判定後に短いクールダウン期間を設ける

### B. キャラクター育成システム

#### 進化段階

| 段階 | 名前 | 必要累計経験値 | 日数目安（1日1曲） |
|------|------|-------------|-----------------|
| 1 | タマゴ | 0 | 0日目 |
| 2 | 赤ちゃん | 3,000 | 1日目 |
| 3 | 子ども | 10,000 | 3日目 |
| 4 | 大人 | 25,000 | 5日目 |
| 5 | マッチョ | 50,000 | 8日目 |
| 6 | ボディビルダー | 85,000 | 11日目 |
| 7 | ゴリラ | 130,000 | 14日目 |

※ 上記は1日1曲・平均スコア5,000点前後（Perfect多め）で約2週間でゴリラ到達する設計。
※ 1曲のノーツ数やスコア上限は難易度・曲により異なるが、目安として1曲最大スコア約10,000点。

#### 経験値計算

```
獲得経験値 = ゲームスコア × 難易度倍率

難易度倍率:
  Normal: 1.0
  Hard:   1.5
```

#### 衰退システム

- **3日間連続**でトレーニングしなかった場合、経験値が減少する
- 減少ルール:
  - 3日目以降、1日あたり累計経験値の **5%** が減少
  - ただし現在の進化段階の下限値を下回らない（退化は最大1段階まで）
  - アプリ起動時に最終プレイ日をチェックし、未プレイ日数に応じて一括計算
- 退化時にはキャラクターの見た目が1段階戻る

#### キャラクターアニメーション

- ホーム画面で待機モーション（アイドルアニメーション）を表示
- 進化段階ごとに異なるアニメーション
- アニメーション実装: スプライトシート or Lottie

## データモデル

### ローカルDB構造

#### training_records テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | INTEGER (PK) | レコードID |
| song_id | TEXT | 曲ID |
| difficulty | TEXT | "normal" / "hard" |
| score | INTEGER | ゲームスコア |
| exp_earned | INTEGER | 獲得経験値 |
| perfect_count | INTEGER | Perfect判定回数 |
| great_count | INTEGER | Great判定回数 |
| good_count | INTEGER | Good判定回数 |
| miss_count | INTEGER | Miss判定回数 |
| max_combo | INTEGER | 最大コンボ数 |
| played_at | DATETIME | プレイ日時 |

#### character_state テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | INTEGER (PK) | 常に1（シングルトン） |
| total_exp | INTEGER | 累計経験値 |
| evolution_stage | INTEGER | 現在の進化段階（1-7） |
| last_played_at | DATETIME | 最終プレイ日時 |
| created_at | DATETIME | 初回起動日時 |
